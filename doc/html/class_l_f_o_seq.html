Detailed Description

Derived class from EnvSegSeq for specific LFO purposes.

The main difference between EnvSegSeq and LFOSeq is that LFOSeq has a rate parameter that sets the rate or frequency of the entire Envelope sequence. Internally, changing this "rate" is equivalent to appropriately adjusting the length of each EnvSeg.

void LFOSeq::setRate ( double  Hz)

Sets the rate of the envelope sequence.

Setting the rate of the sequence is equivalent to appropriately adjusting the length of each individual envelope segment. Given a frequency f, the period or in this case the length of the sequence is 1/f. This value is then divided by the number of segments to give the length of an individual segment, which is then adjusted for each segment.

Parameters
HzThe rate/frequency to set, in Hz.

void LFOSeq::setSidechain_Seg ( seg_t  segNum,
index_t  dockNum,
index_t  master,
index_t  slave 
)
virtual

Makes a ModUnit in a segment's ModDock sidechain another's depth.

Sidechaining means that one ModUnit sidechains/modulates another's depth value. Multiple mastery as well as multiple slavery is possible, so you can call setSidechain for any two indices you wish as long as the connection isn't already established, in which case nothing happens. Note that a ModUnit cannot simultaneously be a master of slaves and contribute to the ModDock's modulation value, it is either-or.

Parameters
segNumThe segment index in the sequence to do this for.
dockNumThe index of the ModDock in which to connect the two ModUnits.
masterThe index of the ModUnit in the ModDock to be made master.
slaveThe index of the slave in the ModDock.
